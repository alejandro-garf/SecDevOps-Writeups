# TryHackMe Room Writeup — Software Development Lifecycle (SDLC)

---

## What is SDLC?

The Software Development Lifecycle (SDLC) is a set of practices that make up a framework to standardise the building of software applications.

**Question Answers:**
- The SDLC can have **6–8 phases**.

---

## SDLC Phases Pt. 1

### Planning

The planning stage can be crucial if the goal is to have a product ready for market by a specific date. This stage is also referred to as the **feasibility stage**.

### Requirements Definition

It is common for developers to produce a document known as a **Software Requirements Specification (SRS)** during this phase. This is also where the first idea for a prototype is determined.

### Design and Prototyping

Defining a platform means choosing which media will run the software. A document called an **Architecture Design Review (ADR)** is typically created by engineers and developers at this stage to ensure that all teams working in different areas are aligned.

**Question Answers:**
- The phase that determines the first idea for a prototype is **Requirements Definition**.
- The feasibility stage refers to the **Planning** stage.
- **Design and Prototyping** is the third phase.

---

## SDLC Phases Pt. 2

### Testing

During the testing stage, developers review their software using automated tooling such as source code scanners to ensure it meets the standards defined in the requirements phase.

### Deployment

The focus during deployment is on automating software rollouts so that teams can deploy new applications to all machines or just selected ones.

### Operations and Maintenance

Operations teams work closely with developers to provide on-demand access to secure, compliant tooling and environments. This phase handles issues and bugs reported by end-users.

**Question Answers:**
- **Operations and Maintenance** is the phase focused on handling issues and bugs reported by end-users.
- **Deployment** is when new versions of software are released.
- **Testing** is how you ensure the software meets the standards defined by the requirements phase.

---

## CALMS Framework

CALMS is a framework that assesses a company's ability to adopt DevOps processes. A large part of DevOps is the focus on automation.

**Question Answers:**
- **CALMS** stands for **Culture, Automation, Lean, Measurement, and Sharing**.

---

## DevOps Metrics

Key DevOps metrics to track include:

- **Mean Time to Production (MTTP)** — The turnaround time for newly committed source code to reach production.
- **Failure Rate** — Tracking and reporting failure rates is essential for ensuring new code releases meet security requirements.
- **Mean Time to Recovery (MTTR)** — How long it takes to recover from a failure.

**Question Answers:**
- **Deployment speed and frequency**
- **Failure rate**
- **MTTR**
